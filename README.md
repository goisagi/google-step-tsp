# WEEK5課題 
プログラムはweek5_hw.py

## 課題の概要
TSPの実装  
必須課題：貪欲法＋2-optを実装  
発展課題：自分ができる範囲で色々トライしてスコアを更新する  

## 使用したアルゴリズム
1. 貪欲法  
現時点で一番近い場所に移動することを繰り返す手法。  
すでに訪れたことがある場所をvisitedで管理し、現在の点から、visitedがFalseになっている点の距離を順番に見ていき、その中で一番距離が近かったものを次の点とする。

2. 2-opt  
ふたつの道が交差している時、それを交換することで全体のルートを短くするという手法。  
このプログラムでは、交差しているかの判定をするのではなく、ある2組の連結を入れ替えた場合に、距離が短くなるかで判定を行なっている。  
前から順番に全ての2組の連結の組み合わせを見ていく。最後まで見終わった後は、そこまでの変更によって、新たに交換すると距離が短くできる箇所が生まれている可能性があるので、再び前から順番に見ていく。  
improvedという更新の有無を表す変数を用意し、最後まで見終わった際に、1度も連結の変更が行われなかった場合はimproved = Falseとなり、ループが終了するようになっている。  
連結の交換（bとcを入れ替え）を行う際は、b〜cの範囲の順序を逆順にすることで実現している。

3. 焼きなまし法  
2-optでは、改善が見られる場合しか交換が行われないので、悪化する場合でも確率的にそれを受け入れることで、2-optのみの時では見つからないような可能性を探索する。  
温度の初期値を1000とし、1回交換を試みるたびに温度を下げていく。  
点の交換を試みる連結は、ランダムで決めている。  
交換によって距離が縮まる場合は、無条件にそれを受け入れる。距離が長くなってしまう場合は、exp(-delta/T)が、ランダムで出した値より大きいならば、それを受け入れるようにしている。  
更新があるたびに、今までで総距離が一番短かった時と比較し、改善されている場合はそれを更新する。  
最終的には、更新作業を行なった中で、一番総距離が短かった時のルートを返す。

main関数では、貪欲法 → 2-opt → 焼きなまし法 → 2-opt の順番で実行し、最終的なルートをファイルに書き込んでいる。  
最後に2-optをもう一度行なったのは、焼きなまし法の工程によって、再び交差している箇所が発生した可能性があるからである。
